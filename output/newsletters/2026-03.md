# Aptos Developer Monthly — 2026-03

> Your monthly digest of Aptos ecosystem updates

Welcome to the March 2026 Aptos Developer Newsletter. This month brings significant updates including a 20x increase in default gas limits across TypeScript and Python SDKs (from 100k to 2M and 1M respectively), improved wallet adapter TypeScript compatibility with fixes for cross-chain derived wallet registration, and enhanced Create Aptos Dapp CLI with non-interactive scaffolding flags for automated workflows. Additionally, AIP-137 proposes post-quantum cryptographic security through SLH-DSA-SHA2-128s signatures, while community contributions include receiver function improvements in Move compiler v2 and major storage infrastructure optimizations.

## Community Highlights

## Aptos Community Highlights Summary

### Compiler & Move Language Improvements
**Receiver Functions Enhancement**: The Move compiler v2 now allows receiver functions (dot-call syntax) on fully concrete struct instantiations, removing the previous requirement for fully generic identity instantiation. Type inference has been improved to allow struct constraints and receiver function constraints to coexist without canceling each other out.

### Infrastructure & Storage Optimization
**Storage Cleanup**: Major storage infrastructure improvements include removing deprecated StateValueSchema and legacy monolithic ledger DB support, as dead code cleanup eliminates unnecessary `#[allow(dead_code)]` suppressions. The deployment system now supports multiple Public Full Nodes (PFNs) via separate helm releases instead of individual Kubernetes resources.

### Gas & Economic Changes
**Gas Cost Adjustment**: A significant 10x multiplier has been applied to gas costs and storage fees, with backports across multiple release versions (v1.41-v1.43). This represents a major economic parameter change for the network.

### Consensus & Security Updates
**Randomness System Simplification**: The consensus layer has been streamlined by removing randomness fast path code, eliminating ~800 lines of complex fast-share reconstruction logic. DKG (Distributed Key Generation) components received major refactoring with simplified sigma protocol code.

### Developer Tools & Testing
**Testing Infrastructure**: New Move unit test generation capabilities have been added to move-flow, including coverage tracking and baseline establishment. The Forklift tool introduces network forking and smart contract testing capabilities for Aptos developers.

### Network & Performance
**Priority Peer Support**: Network infrastructure now supports priority inbound peers that can connect even when connection limits are reached, with automatic eviction of regular peers when needed. Move harness has been separated to avoid unintended "testing" feature activation in production services.

### Strategic Communications
**Vision & Roadmap**: Multiple blog posts outline Aptos's 2025 strategy, including the vision for a "Global Trading Engine," composable exchange infrastructure, and censorship-resistant consensus mechanisms. The Shelby early access program is now available on testnet, representing advances in lazy loading and composable Move architecture.
