# Aptos Developer Monthly — 2026-02

> Your monthly digest of Aptos ecosystem updates

The February 2026 Aptos developer newsletter covers significant infrastructure changes and protocol enhancements. Key breaking changes include the extraction of framework natives into separate `aptos-framework-natives` and `aptos-release-bundle` crates, requiring import path updates for native functions and framework bundles. Major protocol updates include new struct/enum visibility modifiers with bytecode verification, optimistic randomness share verification, and the introduction of ChunkyDKG integration with epoch management for improved distributed key generation.

---

## Breaking Changes

# Aptos Breaking Changes Summary

## Framework Natives Extraction (#18788)

**What broke:**
- `aptos-framework` crate split into separate `aptos-framework-natives` and `aptos-release-bundle` crates
- Import paths for native functions and framework bundles changed
- Dependency structure reorganized

**Why it changed:**
- Reduce rebuild cascades - framework previously depended on entire Move compiler stack
- Compiler/prover changes no longer trigger unnecessary framework rebuilds
- Cleaner separation between runtime natives and build tooling

**Migration required:**
- Update imports from `aptos-framework` to `aptos-framework-natives` for native function references
- Update imports to `aptos-release-bundle` for framework bundle access
- Update `Cargo.toml` dependencies to use new crate names
- Review build scripts that reference framework components

## Protocol Updates

# Aptos Protocol Updates Summary

## Virtual Machine (VM) & Execution

**Security & Validation Improvements**
- Fixed bounds checking and recursion issues in VM with timed feature flags for struct/function/script limits
- Enhanced type depth tracking and async runtime checks to prevent execution/replay divergence
- Added runtime stack size validation for struct unpacking and closure calls to prevent stack corruption
- Improved cryptographic native error handling by removing unwrap() usage

**Performance Optimizations**
- Implemented `Arc<MoveStructLayout>` for shared struct layout caching during construction
- Fixed cache flush mechanism to ensure full cache clearing when capacity reached

## Move Language & Compiler

**New Language Features**
- Added visibility modifiers for structs/enums with bytecode verification (steps 3-4)
- Enabled compiler warnings for unused private functions, structs, and constants
- Added format string validation for assert macros
- Fixed parsing of signed integer returns in explicit return statements

**Developer Tooling**
- Introduced `aptos-move-flow` crate with AI tool integration and MCP stdio server
- Implemented behavioral predicates in Move Prover for higher-order function verification
- Converted MVIR tests to MASM format (removing legacy Move IR dependency)

## Consensus & Randomness

**Randomness Improvements**
- Added optimistic randomness share verification with batch verification at aggregation time
- Deferred expensive cryptographic aggregation for blocks without randomness transactions
- Integrated decryption pipeline and secret share management for encrypted transactions

**Payload Management**
- Consolidated to use `OptQuorumStore` payload exclusively in proof_manager
- Deprecated legacy Payload enum variants (`InQuorumStore`, `InQuorumStoreWithLimit`, etc.)

**DKG (Distributed Key Generation)**
- Added `ChunkyDKGManager` with state machine for DKG coordination
- Integrated ChunkyDKG with epoch manager for reconfiguration
- Added `ValidatorTransaction::ChunkyDKGResult` variant with consensus validation hooks

## Storage & Database

**Performance & Monitoring**
- Added RocksDB ticker statistics reporting via Prometheus for per-DB metrics
- Set `fill_cache(false)` on pruner RocksDB reads to prevent cache pollution
- Refactored `StateKvDb` to separate hot/cold instances matching `StateMerkleDb` pattern

**Bug Fixes & Stability**
- Fixed `HotState` race condition using RCU/deferred merge approach
- Added stale JMT node cleanup thread
- Relaxed panicking assertions in `StateSummary::update` to handle fork scenarios
- Removed dead storage APIs and non-sharding code paths

## Networking

**Architecture Changes**
- Replaced network health checker with peer monitoring service for disconnect logic
- Removed subscription logic from `PeersAndMetadata` (legacy cleanup)
- Removed old netbench code and legacy networking components

**State Sync Optimization**
- Combined 4 separate tokio runtimes into one shared runtime (reducing from 176 to 44 threads on 44-core machines)

## Impact for Node Operators

**Required Updates**
- Validators and fullnodes should upgrade to v1.41.5+ 
- Expect temporary disk usage increase during stale data cleanup for pre-October nodes
- New RocksDB metrics available for monitoring storage performance

**Breaking Changes**
- Storage sharding now always enabled (non-sharding code removed)
- Legacy payload types deprecated in consensus
- Some dead storage APIs removed

**New Features:**
- Added support for spec pq functionality

**Bug Fixes:**

## Community Highlights

# Aptos Ecosystem Community Highlights

## Core Infrastructure & Performance

**Forge Observability Enhancement**: Added Prometheus metrics for cluster spin-up phases and transaction emitter to improve test execution monitoring and debugging capabilities.

**Thread Management Optimization**: Named all unnamed threads in aptos-node execution path to improve debugging through better stack traces and logging, plus reduced default thread counts for backup and index-db runtimes from 44 to 2 threads each.

**Memory Management Tuning**: Configured jemalloc with per-CPU arenas, 2MB huge pages, and background maintenance threads for better performance characteristics.

**Docker Infrastructure Modernization**: Migrated from Debian Bullseye to Trixie and split Docker builds into 4 parallel BuildKit stages (node-builder, forge-builder, tools-builder, indexer-builder) for faster CI/CD.

## Cryptography & Security

**Encrypted Transaction Support**: Implemented end-to-end encrypted transaction lifecycle including API headers, SDK builders, VM execution, and DKG encryption key derivation with comprehensive smoke testing.

**Shplonked PCS Integration**: Added Shplonked polynomial commitment scheme with major refactoring of sigma protocol code, MSM optimizations, and improved PVSS trait design.

**Chunky DKG Implementation**: Added Move modules for Chunky DKG state management, reconfiguration support, and VM integration alongside existing DKG flows.

**Batch Encryption Optimization**: Refactored `aptos-batch-encryption` to work on individual ciphertexts and improved chunky dealing speed with custom `arkworks::batch_mul()` avoiding premature normalization.

## Developer Tools & CLI

**Move CLI Extraction**: Extracted `aptos move` subcommands into standalone `aptos-move-cli` crate for independent building and testing, introducing `DiagWriter` for compiler diagnostics.

**Gas Profiler UI Overhaul**: Delivered major UI redesign with interactive features, sortable/exportable tables, execution trace view, and better alignment with on-chain `FeeStatement`.

**Transaction Simulation Sessions**: Added `new_block` and `advance_epoch` capabilities with gas profiling support for testing block prologue behavior and epoch transitions.

**Transactional Test Improvements**: Added signed integer support in test arguments, fixing clap parsing issues with negative values.

## Framework & VM Enhancements

**ABI Enhancement**: Added enum variant descriptions to node API ABI responses, improving Move module introspection capabilities.

**BlockSTM Pre-write Optimization**: Implemented pre-write timestamps in MVHashMap to reduce contention for frequently accessed resources during parallel execution.

**Emergency Validator Set Protection**: Added safeguards preventing empty validator sets during epoch transitions, ensuring network liveness in extreme staking scenarios.

**Native Function Support**: Enhanced assembler-disassembler with native function roundtripping and improved MASM format adoption across the codebase.

## Code Quality & Maintenance

**Legacy Code Cleanup**: Removed deprecated in-node indexer code, node-checker service, and move-errmapgen crate as functionality has been superseded by newer implementations.

**Tool Reorganization**: Moved move-docgen and move-abigen from move-prover to tools directory, removed move-cli crate entirely, and eliminated build.rs from aptos-cached-packages for faster builds.

**Rust Toolchain Updates**: Upgraded to Rust 1.91 and 1.92 with mechanical lint fixes and dependency updates across the workspace.

**MVIR to MASM Migration**: Converted all .mvir file usage to .masm format using automated tooling with verification of every conversion.
