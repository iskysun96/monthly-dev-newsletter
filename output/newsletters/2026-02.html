Breaking Changes

# Aptos Breaking Changes Summary

## Framework Natives Extraction (#18788)

**What broke:**
- `aptos-framework` crate split into separate `aptos-framework-natives` and `aptos-release-bundle` crates
- Direct imports of native functions from `aptos-framework` no longer work
- Build dependencies restructured to separate tooling from runtime components

**Why it changed:**
- Reduce rebuild cascades when Move compiler/prover changes
- `aptos-framework` previously bundled both build tooling and runtime natives, causing unnecessary rebuilds
- Improves build performance and dependency isolation

**Migration required:**
- Update `Cargo.toml` dependencies:
- Replace `aptos-framework` with `aptos-framework-natives` for runtime native functions
- Add `aptos-release-bundle` for framework release artifacts
- Update import statements:
```rust
// Before
use aptos_framework::natives::*;

// After
use aptos_framework_natives::*;
```
- Review custom native function integrations for new crate structure
- Rebuild projects to resolve dependency conflicts

Protocol Updates

# Aptos Protocol Updates Summary

## Virtual Machine & Compiler

**VM Security & Validation**
- Fixed bounds checking and recursion issues in VM bytecode verification (PR #18887) - **Medium Risk**: Changes production bytecode verification limits behind timed feature flag
- Enhanced stack size validation for struct unpacking and closure calls to prevent stack corruption (PR #18630)
- Improved type depth tracking and async runtime checks to prevent execution/replay divergence (PR #18728)

**Move Language Features**
- Added struct/enum visibility modifiers (PRs #18117, #18165) - implements bytecode verification for new visibility rules
- Enabled compiler warnings for unused private functions, structs, and constants (PR #18661)
- Fixed signed integer return parsing in compiler v2 (PR #18780)
- Added format string validation for assert macros (PR #18533)

**Developer Tools**
- New `aptos-move-flow` crate with AI tool integration and MCP stdio server for Move development (PR #18823)
- Move Prover now supports function values/closures with behavioral predicates (PR #18465)
- Migrated VM transactional tests from .mvir to .masm format (PR #18510)

## Storage & Performance

**Storage Optimizations**
- Refactored StateKvDb to separate hot/cold instances matching StateMerkleDb pattern (PR #18577)
- Fixed HotState race condition using RCU/deferred merge pattern (PR #18744)
- Set `fill_cache(false)` on pruner RocksDB reads to prevent cache pollution (PR #18795)
- Added RocksDB ticker statistics reporting via Prometheus (PR #18878)
- Started cleanup thread for stale JMT nodes (PR #18688)

**Execution & Caching**
- Fixed cache flush behavior when caches are full (PR #18884)
- Implemented `Arc` for shared struct layout caching in VM (PR #18640)

## Consensus & Networking

**Consensus Improvements**
- Deferred randomness aggregation for non-randomness blocks to reduce crypto overhead (PR #18699)
- Added optimistic randomness share verification with batch verification (PR #18646)
- Consolidated payload generation to use `OptQuorumStore` exclusively (PR #18653)
- Deprecated old Payload enum variants (`InQuorumStore`, `InQuorumStoreWithLimit`, etc.) (PR #18578)

**DKG Integration**
- Added chunky DKG types, manager, and epoch manager integration (PRs #18542, #18544, #18545)
- Added `ValidatorTransaction::ChunkyDKGResult` variant with consensus hooks (PR #18539)
- Implemented decryption pipeline and secret share management (PR #18758)

**Network Stack**
- Replaced network health checker with peer monitoring service for disconnect logic (PR #18790)
- Removed legacy subscription logic from PeersAndMetadata (PR #18798)
- Removed old netbench code (PR #18789)
- Consolidated 4 state-sync tokio runtimes into single shared runtime, reducing thread count from 176 to ~44 on 44-core machines (PR #18781)

## Releases

**Node Releases**
- v1.41.5: Includes stale data cleanup (validators and fullnodes should upgrade)
- v1.40.4: Security fixes for VM cryptographic natives and transaction size checks
- Multiple hotfixes (v1.40.2, v1.40.3, v1.40.4) - use Docker images/binaries, not source builds

**CLI Releases**
- v8.1.0, v8.0.0, v7.14.2: Various improvements and license updates

## Impact Summary

**Node Operators**: Storage optimizations and network improvements reduce resource usage

## PVSS Aggregation Type Enhancement

**Feature**: New `Aggregated` type for PVSS (Publicly Verifiable Secret Sharing) aggregation in aptos-dkg

**Developer Impact**:
- Improves performance and simplifies code in distributed key generation protocols
- Enables more efficient handling of transcript data by storing affine elements instead of projective elements
- Optimizes aggregation operations in the `chunky` implementation

**Technical Details**: The enhancement addresses performance bottlenecks in naive `aggregate_with()` usage while maintaining the security properties of the PVSS scheme.

**Reference**: [PR #18514](https://github.com/aptos-labs/aptos-core/pull/18514)

Community Highlights

# Aptos Ecosystem Community Highlights

## Core Infrastructure & Performance

**Forge Observability Enhancement**: Added Prometheus metrics for cluster spin-up phases and transaction emitter to improve test execution monitoring and debugging capabilities.

**Thread Management Optimization**: Named all unnamed threads in aptos-node execution path for better debugging, and reduced default thread counts for backup/index-db runtimes from 44 to 2 threads on 44-core machines.

**Memory Management Tuning**: Configured jemalloc with per-CPU arenas, 2MB huge pages, and background maintenance threads for improved performance, plus added periodic jemalloc stats export as Prometheus gauges.

## Codebase Modernization

**Legacy Code Removal**: Deleted deprecated in-node indexer code (replaced by indexer-grpc service) and removed node-checker tooling entirely from the ecosystem.

**CLI Restructuring**: Extracted `aptos move` commands into standalone `aptos-move-cli` crate for independent building/testing, and removed obsolete `move-cli` crate.

**Move Toolchain Cleanup**: Relocated move-docgen and move-abigen from move-prover to tools directory, removed move-errmapgen crate, and converted all .mvir files to .masm format.

## Encrypted Transactions & DKG

**Encrypted Transaction Support**: Added end-to-end encrypted transaction lifecycle including API headers, SDK builders, VM execution, and emitter support with smoke test validation.

**Chunky DKG Implementation**: Introduced chunky DKG Move modules, reconfiguration support, and batch encryption optimizations for individual ciphertext processing.

**Cryptographic Enhancements**: Redesigned PVSS traits, improved chunky verifier speed with custom batch multiplication, and added RNG parameters to verification methods.

## Developer Experience

**Gas Profiler Overhaul**: Delivered major UI redesign with interactive reports, sortable tables, execution trace views, and better alignment with on-chain FeeStatement.

**Transaction Simulation Sessions**: Added `new_block` and `advance_epoch` capabilities plus gas profiling support for comprehensive simulation testing.

**Testing Improvements**: Added signed integer support in transactional test arguments and enhanced Move framework with abort messages for native functions.

## API & Type System

**Node API Enhancement**: Added enum variant descriptions to ABI responses returned by `/v1/accounts//module/` endpoints.

**Transaction Hash Refactoring**: Introduced `committed_hash()` method on Transaction type and removed CryptoHash trait to enforce proper usage patterns.

**Framework Updates**: Added BlockMetadataExt V2 with decryption key support and emergency validator set protection to prevent empty validator scenarios.

## Build & Deployment
