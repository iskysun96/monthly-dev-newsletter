2026-03
The March 2026 Aptos developer newsletter highlights the release of **Forklift**, a new network forking framework that enables testing smart contracts against real mainnet state, alongside significant SDK improvements including 10-20x gas limit increases in both TypeScript and Python SDKs. Additional updates include Move compiler v2 enhancements for receiver functions, wallet adapter bug fixes for derived wallets, and new CLI flags for Create Aptos Dapp to support AI-friendly non-interactive scaffolding.

SDK & Tooling

**Aptos TypeScript SDK** received a significant gas limit increase, raising the default max gas from the previous limit to 2,000,000. This change provides more headroom for complex transactions and reduces the likelihood of gas-related failures.

**Aptos Python SDK** similarly increased its default `max_gas_amount` in `ClientConfig` from 100,000 to 1,000,000 (10x increase). Developers using the default configuration will automatically benefit from the higher gas limits without code changes.

**Aptos Wallet Adapter** released several updates across multiple packages:

• **@aptos-labs/wallet-adapter-react@8.3.1** - Fixed JSX intrinsic element typing issues by importing `JSX` and `ClassAttributes` directly from React, replacing global `JSX.IntrinsicElements` references with `ReactJSX.IntrinsicElements`. This resolves TypeScript compilation issues in certain React/TypeScript configurations.

• **@aptos-labs/wallet-adapter-core@8.5.0** - Updated the Nightly wallet logo.

• **Derived wallet packages** (ethereum@0.9.1, solana@0.12.2, sui@0.2.1) - Fixed a critical bug in `setupAutomaticDerivation` where old derived wallets weren't properly unregistered when providers re-announced with the same identifier. This prevents stale wallet instances from remaining in the registry.

• **@aptos-labs/wallet-adapter-mui-design@5.5.4** and **@aptos-labs/wallet-adapter-ant-design@5.3.17** received dependency updates.

**Create Aptos Dapp** added CLI flags for non-interactive and hybrid scaffolding modes. This enhancement makes the tool AI-friendly by allowing template scaffolding through single CLI commands instead of requiring interactive prompts. The hybrid approach supports partial flags while falling back to interactive mode for missing parameters.

Protocol Updates

The latest Aptos protocol updates include significant improvements to Move language features, storage optimizations, consensus simplifications, developer tooling enhancements, and networking improvements.

**Move Compiler Enhancements**

The Move compiler v2 received two important updates that expand receiver function capabilities. Developers can now use receiver functions (dot-call syntax with `self`) on fully concrete struct instantiations, removing the previous restriction that required generic identity instantiation. This enables patterns like `fun foo(self: S)` that were previously blocked. Additionally, type inference improvements now allow struct constraints and receiver function constraints to coexist without canceling each other out, reducing the need for explicit type annotations in many cases.

**Storage Layer Cleanup**

Node operators will benefit from storage layer optimizations that remove legacy components. The deprecated `StateValueSchema` and its associated column family have been completely removed, as all state value operations now use the more efficient sharded `StateKvDb` through `StateValueByKeyHashSchema`. The monolithic `LedgerDb` architecture has been fully decomposed into individual sub-databases. These changes also include cleanup of dead code suppressions throughout the storage module, replacing blanket `#[allow(dead_code)]` attributes with proper conditional compilation flags for test-only code.

**Consensus Simplification**

Validators running consensus nodes will see reduced complexity with the removal of randomness fast path code. This eliminates approximately 800 lines of code across 29 files, including `ConfigV2`, `PathType`, `FastShare`, and related broadcast mechanisms. The fast path, which used separate higher secrecy thresholds for faster randomness reconstruction, was disabled on mainnet and has now been completely removed from the codebase.

**Developer Tooling**

dApp developers gain new testing capabilities through Move Flow's unit test generation support. The MCP server now includes two new tools: `move_package_test` for running Move unit tests with optional baseline establishment for coverage tracking, and functionality to report newly covered or uncovered lines in subsequent test runs. This enhances the development workflow for Move-based applications.

**Network Infrastructure**

Node operators can now configure priority inbound peer support, allowing specified peers to connect even when inbound connection limits are reached. When priority peers attempt to connect to a full node, the system will evict existing non-priority connections to accommodate them. This feature improves network reliability for critical peer relationships in validator and full node deployments.

New Features

Aptos has introduced **Forklift**, a new network forking and smart contract testing framework that enables developers to create isolated testing environments by forking the mainnet state. This tool allows developers to test smart contracts against real network conditions and data without affecting the live network.

Key capabilities for developers include:

• **Mainnet state forking** - Create test environments with actual on-chain data, enabling realistic testing scenarios with existing protocols, accounts, and contract states

• **Smart contract testing** - Deploy and test contracts in forked environments to validate behavior against real market conditions and user data before mainnet deployment

• **Integration testing** - Test complex interactions between multiple protocols and contracts using live network state as the foundation

• **Debugging and simulation** - Reproduce mainnet issues in controlled environments for debugging and optimization

Forklift addresses common developer pain points around testing smart contracts in realistic conditions, moving beyond synthetic test data to enable validation against actual network state. This is particularly valuable for DeFi protocols, NFT marketplaces, and other applications that need to interact with existing on-chain ecosystems.

The framework integrates with existing Aptos development tooling and supports standard testing workflows. Developers can specify block heights for forking, manipulate account states, and run comprehensive test suites against forked networks.

Full details and implementation guidance are available in the [announcement blog post](https://medium.com/aptoslabs/introducing-forklift-network-forking-and-smart-contract-testing-for-aptos-d516b815d625?source=rss-70211828fe2e------2).

Governance & AIPs

**New AIPs**

• AIP-137: Post-quantum Aptos accounts via SLH-DSA-SHA2-128s signatures - Status: New - Proposes adding support for post-quantum cryptographic signatures using the SLH-DSA-SHA2-128s algorithm to enable quantum-resistant account authentication on Aptos.

This proposal introduces quantum-resistant cryptography to future-proof account security against potential quantum computing threats. Developers should evaluate the performance implications of post-quantum signatures, which typically have larger signature sizes and different computational requirements compared to current ECDSA implementations. The change would require updates to wallet integrations and any applications that handle signature verification directly.

Community Highlights

The Aptos ecosystem released several significant blog posts covering major developments in 2025. The Move smart contract language received substantial improvements including lazy loading capabilities that enhance composability and modularity. A new censorship-resistant consensus mechanism was introduced to enable fairer market operations.

Shelby, a new platform or tool, launched early access on Aptos testnet for developer experimentation. The team outlined their vision for a "Composable Global Exchange Engine" and shared infrastructure improvements that bring Aptos closer to becoming a global trading platform.

• **Move Language Evolution**: Lazy loading functionality enables more composable Move contracts, allowing developers to build more modular applications with improved resource efficiency.

• **Consensus Improvements**: New censorship-resistant consensus mechanisms provide stronger guarantees for fair market operations and transaction ordering.

• **Testnet Access**: Shelby early access on testnet gives developers hands-on experience with new platform capabilities before mainnet deployment.

• **Infrastructure Scaling**: Helm-based deployment improvements allow multiple Public Full Nodes (PFNs) to be deployed as separate releases, improving node management flexibility for operators.

• **Developer Tooling**: Move harness separation removes unintended testing dependencies from production builds, preventing hardcoded test configurations from affecting live services.

• **Cryptographic Updates**: DKG (Distributed Key Generation) protocol refactoring simplifies sigma protocol implementations and removes deprecated proof systems.

The 2025 recap highlighted Aptos Labs' progress toward building financial infrastructure, with particular emphasis on developer experience improvements and protocol optimizations that support high-throughput trading applications.
